# web 优化

## 优化目的

- 从用户角度而言，优化能够让页面加载得更快、对用户的操作响应得更及时，能够给用户提供更为友好的体验。
- 从服务商角度而言，优化能够减少页面请求数、或者减小请求所占带宽，能够节省可观的资源。

| class      | 途径                                |
| ---------- | ----------------------------------- |
| 设计       | 简化页面                            |
| HTTP       | 合理设置 HTTP 缓存                  |
| ^          | 减少不必要的 HTTP 跳转              |
| 打包       | css 雪碧图 + 小图片 base64          |
| ^          | 摇树                                |
| ^          | 按需加载                            |
| ^          | 资源名扩展： chunkhash contenthash  |
| ^          | 代码分割                            |
| ^          | service workers 实现离线页面（PWA） |
| 代码级优化 | 减少不必要的 DOM 操作               |
| ^          | javascript 内存释放                 |
| 安全       | 用户输入内容转义                    |
| ^          | 反爬虫策略                          |
| SEO        | 语义化标签                          |
| ^          | 服务端渲染                          |
| ^          | <meta> attr 等                      |
